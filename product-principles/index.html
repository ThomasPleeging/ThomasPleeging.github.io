It’s the UX, not the UI

We don’t consider a feature done until it makes growers more effective. What we ship only truly matters when there is a holistic experience around it. It’s the UX, not the UI. Service + Product = UX

two

We set the industry standard in cultivation management

So we are not going to do weird metrics.

three

Trust is built on data quality

Bad data is worse than no data. We need good data to train AI models. Growers can't make decisions on bad data.

four

Opinionated on fundamentals, flexible on usage

Our fundamentals are firmly rooted in plant and data science. We know this best. Growers know their job best, and so we are flexible in how they can use the product. There is no wrong way to use Source, but there is a default way.

five

We create a knowledge multiplier for growers

Over time, less and less input required, and more and more output delivered.

six

New features start small first version and we make pragmatic decisions

We build features in incremental steps, validating troughout. We choose to release value as fast as possible.

seven

We strive for operational dependency

Users start and end their day with Source. Every feature should contribute to this. Growers wake up and go to bed with Source.
